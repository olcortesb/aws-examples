## Methodology

1. Each Lambda is invoked 100 times sequentially via Function URL
2. Each invocation executes the `all` operation (write → list → read → delete)
3. Elapsed time (ms) is measured inside the Lambda for each operation using `time.time()`
4. Statistics are calculated from the collected times: avg, min, max, median, P95
5. Both Lambdas use the same S3 bucket and 512 MB memory
6. S3 Files Lambda runs inside a VPC with mounted filesystem
7. SDK Lambda runs without VPC using boto3 `s3` client

### How time is measured

Each operation is timed internally inside the Lambda function using Python’s `time.time()` before and after the core operation. This measures only the actual I/O time, excluding Lambda cold start, network latency from the caller, and Function URL overhead.

**S3 Files (filesystem):**
```python
start = time.time()
with open('/mnt/s3files/test.txt', 'w') as f:
    f.write(content)
elapsed_ms = (time.time() - start) * 1000
```

**SDK (boto3):**
```python
start = time.time()
s3.put_object(Bucket=bucket, Key=key, Body=content.encode())
elapsed_ms = (time.time() - start) * 1000
```

The same pattern applies to all operations (read, list, delete). Only the target operation is measured — no setup, serialization, or response formatting is included in the elapsed time.

## Calculation Formulas

```
Average (Avg) = Sum of all times / Number of samples
Median        = Middle value when sorted (50th percentile)
P95           = Value at 95th percentile when sorted
Speedup       = SDK Avg / S3 Files Avg
Difference    = SDK Avg - S3 Files Avg
```

### Example Calculation (write operation)

```
S3 Files Write Avg = 1315.45 / 100 = 13.15 ms
SDK Write Avg      = 3020.15 / 100 = 30.2 ms
Difference         = 30.2 - 13.15 = 17.05 ms
Speedup            = 30.2 / 13.15 = 2.3x
```
